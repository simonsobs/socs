UKKIDcontroller agent - installation and tutorial
=================================================

## Introduction

The UKKIDcontroller agent, ukkid_controller.py is an OCS agent that
communicates with hardware KRM RF System-on-chip (RFSoC) boards which run the
software and firmware for multi-tone readout of the UK Kinetic Inductance Detectors (UKKIDs).

The RFSoC boards feature DACs, ADCs, FPGA fabric and a two-core ARM
processor running Ubuntu Linux. The tones to readout the KIDs are
generated by firmware pipelines running on the FPGA
fabric. Communication with these firmware pipelines is achieved by
python-based "Readout server" software which runs on the RFSoCs ARM
cores. This server software takes IQ data which appears in the
firmwares/ARM cores shared memory and packages it into TCP/IP packets
which are sent to another computer which runs python software called
the "readout client". In practice this "readout client" software is a
python module, which can be imported into a python command line
interpreter and run interactively, or imported into other python
software or scripts. In our case the ukkid_controller.py imports the
readout_client module, which contains various methods for sending
instructions to and receiving data from the readout server saoftware running
on the RFSoc's ARM cores.

 The ukkid_controller.py has processes and tasks with effectively act
 as wrapper functions (with appropriate logging and error) for the
 functions provided by the readout_client.py module. These
 tasks/processes can e.g. request readout server configuration, start
 frequency scans, set tone frquencies and amplitudes and start
 streaming of IQ data. When streaming data for an astonomical
 observation, the ukkid_controller.py does not receive IQ TCP/IP data
 directly, but rather spawns separate streaming python streaming tasks
 (which also use the readout_client.py to communicate with the
 RFSoCs). The design is such that the ukkid_controller.py provides
 both sufficiently high-level tasks such that they can be used
 directly by telescope observers using the python "nextline" tool, and
 also low-level tasks for instrument characterisation and calibration,
 e.g. during in commissioning. The ukkid_controller.py also provides
 tools and interfaces (via the resonator_fitter module) for fitting
 and extracting resonator parameters from freqeuncy scans across the
 KID resonator arrays.


## Installation of prerequisities

 On a standard Ubuntu linux installation (at the time of writing
 Ubuntu 22.04 LTS) the following pre-requisities are required.

1. An Observatory Control System (OCS) installation 

2. souk_readout_tools. This is the python code for the readout client
described above. (Note this repository also contains the code for the
readout server software that runs on the RFSoCs).

3. resonator_fitter. (Double check where this comes from).

4. res_fns.py. Inserted by hand at the moment.

5. The ukkid_controller agent itself - i.e. everything within with the
agents/UKKIDcontroller directory.

We will describe each of these in turn.

### OCS system

For this, one first needs to install docker (including docker compose). The instructions for this can be found here.

https://docs.docker.com/engine/install/ubuntu/#install-using-the-repository

The OCS system can be installed by following the instructions here.

https://ocs.readthedocs.io/en/main/user/installation.html

If installing for the first time, it is useful to read the
documentation about fake-data-agent and work through the "Create your
own agent tutorial".

After installing the OCS and running up the docker agents, one should
confirm that the docker agents are being aluches correctly e.g. you
should get something like

### souk_readout_tools

One should follow the installation instructions here...

https://github.com/sr-cdf/souk_readout_tools

We are in the process of making these instruction clearer, but the procedure that seems to work is.

cd souk_readout_tools/
python3 -m venv client_venv
source ./client_venv/bin/activate
pip3 install -r requirements.txt
pip3 install .
